### 2.1 spaCy (explosion)
- **Objetivo:** NLP industrial em Python com pipelines pré-treinados
- **Stack:** Python + Cython
- **Modelos:** 70+ idiomas, incluindo pt_core_news_lg para português
- **Features:** NER, POS tagging, dependency parsing, text classification
- **Instalação:** `pip install spacy` + download de modelos `python -m spacy download pt_core_news_lg`
- **Uso no Scrapped:** Classificação de intenção (Pessoa, Organização, Localização)
- **Links úteis:** https://spacy.io/models (modelos), https://course.spacy.io (curso)

#### 2.2 sentence-transformers (huggingface)
- **Objetivo:** State-of-the-art text embeddings
- **Stack:** Python + PyTorch + Transformers
- **Modelos:** 15,000+ pré-treinados no Hugging Face
- **Modelos recomendados:** all-MiniLM-L6-v2 (rápido), paraphrase-multilingual-mpnet-base-v2 (multilingue)
- **Instalação:** `pip install sentence-transformers`
- **Uso no Scrapped:** Embeddings semânticos para score de confiança, similaridade
- **Código base:**
```python
from sentence_transformers import SentenceTransformer
model = SentenceTransformer("paraphrase-multilingual-mpnet-base-v2")
embeddings = model.encode(["texto1", "texto2"])
similarities = model.similarity(embeddings, embeddings)
```

#### 2.3 googlesearch-python (Nv7-GitHub)
- **Objetivo:** Library para scraping do Google Search
- **Stack:** Python + requests + BeautifulSoup4
- **Instalação:** `pip install googlesearch-python`

for url in search("eventos Fortaleza", num_results=10, lang="pt"):

#### 4.1 scrapy
- **Objetivo:** Framework assíncrono de scraping web de alto nível
- **Stack:** Python + Twisted (async reactor)
- **Features:** Scrapy Pipeline, Spider middleware, Scrapyrt (API)
- **Instalação:** `pip install scrapy`
- **Arquitetura:** Engine → Spider → Item → Pipeline → Output
- **Scrapyrt:** API REST para controle de spiders
- **Uso no Scrapped:** Scraping de sites estruturados
- **Código base:**
```python
import scrapy
class MySpider(scrapy.Spider):
    name = 'my_spider'
    def start_requests(self):
        yield scrapy.Request(url='https://example.com')
    def parse(self, response):
        pass
```

#### 4.2 playwright (microsoft)
- **Objetivo:** Automação de browser para scraping dinâmico
- **Stack:** Python + Playwright (Chromium, Firefox, WebKit)
- **Instalação:** `pip install playwright` + `playwright install`
- **Features:** espera automática, screenshots, PDF, interceptação de rede
- **Uso no Scrapped:** Páginas que requerem JavaScript (LinkedIn, Google Maps)
- **Código base:**
```python
from playwright.sync_api import sync_playwright
with sync_playwright() as p:
    browser = p.chromium.launch()
    page = browser.new_page()
    page.goto("https://linkedin.com")
    browser.close()
```

### 5. Infraestrutura e Backend

#### 5.1 FastAPI (tiangolo)
- **Objetivo:** Framework web moderno e rápido para APIs Python
- **Stack:** Python + Starlette (ASGI) + Pydantic
- **Instalação:** `pip install fastapi` + `pip install uvicorn[standard]`
- **Features:** Async/await nativo, documentação automática OpenAPI/Swagger, validação de dados com Pydantic, Dependency Injection, WebSockets
- **Uvicorn:** Servidor ASGI recomendado
- **Uso no Scrapped:** API REST principal
- **Código base:**
```python
from fastapi import FastAPI
app = FastAPI()
@app.get("/items/{item_id}")
def read_item(item_id: int):
    return {"item_id": item_id}
```

#### 5.2 authlib (lepture)
- **Objetivo:** Biblioteca OAuth/OAuth2 para Python
- **Stack:** Python pura
- **Instalação:** `pip install authlib`
- **Integração:** Flask, Django, FastAPI, httpx
- **OAuth Providers:** Google, GitHub, Facebook, Twitter
- **FastAPI Integration:** https://docs.authlib.org/en/stable/client/fastapi.html
- **JWT:** Suporte a JSON Web Tokens
- **Uso no Scrapped:** Autenticação OAuth Google
- **Código base:**
```python
from authlib.integrations.starlette_client import OAuth
oauth = OAuth(app)
oauth.register('google', ...)
```

#### 5.3 SQLAlchemy
- **Objetivo:** ORM e toolkit SQL para Python
- **Stack:** Python + Dialects (PostgreSQL, MySQL, SQLite)
- **Instalação:** `pip install sqlalchemy`
- **Modos:** Core (procedural) e ORM (objectos mapeados)
- **Async:** Suporte a asyncio com `sqlalchemy[asyncio]` + `asyncpg`
- **Alembic:** Migrações de banco de dados
- **Uso no Scrapped:** Acesso ao PostgreSQL + SQLite CNPJ

#### 5.4 celery
- **Objetivo:** Task queue distribuído para tarefas assíncronas
- **Stack:** Python + Message Broker (Redis, RabbitMQ)
- **Instalação:** `pip install celery`
- **Broker:** Redis é o mais comum (`pip install redis`)
- **Workers:** Múltiplos workers para processamento paralelo
- **Scheduler:** Beat para tarefas periódicas
- **FastAPI Integration:** `from celery import Celery`
- **Uso no Scrapped:** Scraping paralelo de múltiplas fontes
- **Monitoramento:** Flower (`pip install flower`)

## Notas dos Repositórios Pesquisados

### 1. Receita Federal

#### 1.1 cnpj-sqlite (rictom)
- **Objetivo:** Script Python para converter dados públicos de CNPJ da Receita Federal para SQLite
- **Entrada:** Arquivos CSV zipados do site da Receita Federal
- **Saída:** Arquivo SQLite (cnpj.db) ~30GB
- **Dependencies:** pandas, dask, sqlalchemy, wget
- **Tempo de execução:** ~2h em máquina i7
- **Pré-requisitos:** 60GB disco livre
- **Repositório relacionado:** rede-cnpj (visualização gráfica)

#### 1.2 socios-brasil (turicas/Brasil.IO)
- **Objetivo:** Extrair dados de sócios de empresas brasileiras da Receita Federal
- **Dados disponíveis:** empresa.csv.gz, socio.csv.gz, cnae-secundaria.csv.gz, holding.csv.gz
- **Censura:** Remove dados sensíveis (e-mail, telefones de MEI)
- **Importação:** Suporta SQLite e PostgreSQL via rows CLI
- **Licença:** LGPL-3.0 (código), CC BY-SA 4.0 (dados)
- **Repositório relacionado:** https://brasil.io/dataset/socios-brasil

#### 1.3 rede-cnpj (rictom)
- **Objetivo:** Visualização gráfica de relacionamentos entre empresas e sócios
- **Stack:** Python + Flask + JavaScript (VivaGraphJS)
- **Tabelas necessárias:** cnpj.db, rede.db, rede_search.db, cnpj_links_ete.db
- **Funcionalidades:** Busca por CNPJ/nome, visualização de vínculos, exportação Excel/JSON
- **Versão online:** https://www.redecnpj.com.br
- **Dependências Python:** 3.9 a 3.12

#### 1.4 dados_publicos_cnpj_receita_federal (libercapital)
- **Objetivo:** ETL completo para PostgreSQL via Docker
- **Volume total:** ~20GB (~160 milhões de linhas)
- **Tabelas:** rf_company (~51M), rf_company_root (~47M), rf_partners (~22M), rf_company_root_simples (~30M), rf_company_tax_regime (~10M)
- **Estratégia:** Processamento em chunks de 100k linhas
- **Docker:** docker-compose.yaml com PostgreSQL
- **Update:** Estratégia de tablespace com sufixo _new para zero-downtime
